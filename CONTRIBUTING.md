# Contributing to The QHSE Standard

Thanks for helping improve our work. We are a **Synthetic Media Lab** publishing
content about Quality, Health, Safety, and Environment — so most contributions
are about the *accuracy and clarity of ideas*, not code.

By participating, you agree to abide by our
[Code of Conduct](./CODE_OF_CONDUCT.md).

## Ways to contribute

### 1. Report a factual error or correction
If a published article misstates a fact, misattributes a concept (e.g. Safety-II,
HOP, Human Factors), or contains a broken claim, please
[open a correction issue](https://github.com/the-qthe-standard/.github/issues/new/choose).
Include:
- **Where** — link to the page or repository and section.
- **What's wrong** — quote the exact text.
- **The correction** — what it should say, ideally with a citable source.

### 2. Suggest a topic
Have an incident, paradigm, or question you'd like us to cover? Open a
**Topic suggestion** issue and tell us why it matters to practitioners.

### 3. Fix typos, links, or formatting
Small fixes (spelling, dead links, Markdown) are welcome as pull requests. For
anything larger, please open an issue first so we can align before you invest
time.

## Pull request process

1. Fork the relevant repository and create a branch from `main`.
2. Keep each PR focused on a single change.
3. Describe **what** changed and **why** in the PR body (see the PR template).
4. For content changes, cite your sources so an editor can verify them.
5. A maintainer will review for editorial accuracy before merging.

## Editorial principles

We communicate **established paradigms** in safety science — we do not invent
bespoke operational safety advice. Contributions should be:

- **Sourced** — grounded in recognized literature or primary incident reports.
- **Faithful** — represent original authors and frameworks accurately.
- **Clear** — written for managers, engineers, and leaders, not specialists only.
